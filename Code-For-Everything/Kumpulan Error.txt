List Error Orangepi

1. Could not chdir to home directory /home/insun: No such file or directory
Pesan itu artinya folder home kamu tidak ditemukan di server OrangePi. Ini terjadi karena user insun terdaftar di sistem tapi folder /home/insun nya belum ada atau terhapus.

Cara Mengatasinya
Langkah 1 — Cek Apakah Folder Ada
ls /home
Lihat apakah folder insun ada atau tidak.

Langkah 2 — Buat Folder Home
Jika foldernya tidak ada, buat manual:
sudo mkdir /home/insun

Langkah 3 — Set Kepemilikan Folder
sudo chown insun:insun /home/insun

Langkah 4 — Set Permission Folder
sudo chmod 755 /home/insun

Langkah 5 — Buat Ulang Folder Home Otomatis
Cara lebih lengkap, biarkan sistem yang buat otomatis:
sudo mkhomedir_helper insun

Setelah Itu Logout & Login Ulang
exit

Lalu login kembali, pesan error tersebut seharusnya sudah hilang.

Kenapa Ini Bisa Terjadi?
- Folder terhapusFolder /home/insun tidak sengaja terhapus
- User dibuat manualUser dibuat tanpa otomatis membuat folder home
- Reinstall OSOS diinstall ulang tapi user lama masih terdaftar

2. ntfs-3g-mount: bad mount point home/insun/media/insundrive: No such file or directory
Error ini artinya folder tujuan mount harddisk belum ada. Folder /home/insun/media/insundrive harus dibuat dulu sebelum bisa di-mount.

Langkah 1 — Buat Folder Mount Point
sudo mkdir -p /home/insun/media/insundrive
-p artinya buat semua folder sekaligus meskipun folder parent belum ada

Langkah 2 — Set Kepemilikan Folder
sudo chown insun:insun /home/insun/media/insundrive

Langkah 3 — Coba Mount Ulang
sudo mount /dev/sdX /home/insun/media/insundrive

Ganti /dev/sdX dengan nama harddisk kamu, cek dengan perintah:
lsblk

Langkah 4 — Cek Apakah Berhasil
ls /home/insun/media/insundrive
Jika isi harddisk muncul berarti berhasil ✅

2. Error untuk function formatter

Itu error karena ekstensi intl (Internationalization) PHP belum terinstall. Ekstensi ini diperlukan untuk NumberFormatter. Berikut cara mengatasinya:

Langkah 1 — Install Ekstensi intl
bashsudo apt install php8.1-intl -y

Langkah 2 — Restart PHP-FPM
bashsudo systemctl restart php8.1-fpm

Langkah 3 — Verifikasi Ekstensi Terinstall
bashphp -m | grep intl
Output yang benar:
intl

Langkah 4 — Refresh Browser
Buka kembali website kamu, error 500 seharusnya sudah hilang ✅

Jika Masih Error, Coba Alternatif Kode
Jika tidak ingin install ekstensi, bisa ganti kode tersebut dengan cara manual:
php// Ganti ini
$formatter = new NumberFormatter('en_US', NumberFormatter::PERCENT);

// Dengan ini (tanpa ekstensi intl)
$percentage = number_format($nilai * 100, 2) . '%';

Cek Ekstensi PHP yang Terinstall
Untuk referensi kedepannya, cek semua ekstensi yang aktif:
bashphp -m

Coba install ekstensi intl dulu ya, itu cara paling mudah! 😊

Error Node Exporter

Sama seperti Prometheus tadi, binary Node Exporter juga perlu disesuaikan dengan arsitektur ARM. Download yang sesuai:
Kalau aarch64 (ARM64):
bashcd /tmp
wget https://github.com/prometheus/node_exporter/releases/download/v1.7.0/node_exporter-1.7.0.linux-arm64.tar.gz
tar xvf node_exporter-1.7.0.linux-arm64.tar.gz
cd node_exporter-1.7.0.linux-arm64

sudo cp node_exporter /usr/local/bin/
sudo chown node_exporter:node_exporter /usr/local/bin/node_exporter

sudo systemctl restart node_exporter
sudo systemctl status node_exporter

>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>

Untuk Grafana tidak perlu khawatir soal arsitektur karena instalasinya tadi menggunakan apt package manager yang otomatis mendeteksi arsitektur ARM64 dan mendownload versi yang sesuai.
Jadi perintah yang tadi:
bashsudo apt install -y grafana
Sudah otomatis install versi ARM64. Tidak perlu download manual seperti Prometheus dan Node Exporter.

Langsung cek apakah Grafana sudah berjalan:
bashsudo systemctl status grafana-server
Kalau belum jalan:
bashsudo systemctl enable grafana-server
sudo systemctl start grafana-server

Error Promotheus
Error Exec format error artinya binary Prometheus yang didownload tidak sesuai arsitektur CPU mini servermu. Kemungkinan mini servermu pakai ARM bukan AMD64.

Cek arsitektur CPU dulu:

bash
uname -m
Kemungkinan hasilnya:

x86_64 → AMD64 (Intel/AMD biasa)
aarch64 → ARM64 (Raspberry Pi, dll)
armv7l → ARM 32-bit
Setelah tahu arsitekturnya, download binary yang sesuai:

Kalau aarch64 (ARM64):

bash
cd /tmp
wget https://github.com/prometheus/prometheus/releases/download/v2.51.0/prometheus-2.51.0.linux-arm64.tar.gz
tar xvf prometheus-2.51.0.linux-arm64.tar.gz
cd prometheus-2.51.0.linux-arm64

sudo cp prometheus /usr/local/bin/
sudo cp promtool   /usr/local/bin/
sudo chown prometheus:prometheus /usr/local/bin/prometheus
sudo chown prometheus:prometheus /usr/local/bin/promtool

sudo systemctl restart prometheus
sudo systemctl status prometheus
